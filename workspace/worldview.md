# Worldview

## 对 AI 的基本判断

当前 AI 发展处于：

**大模型能力爆发 + 应用层重构阶段**

主要趋势：

1. Agent 化
LLM 从单轮问答 → 长链路执行系统

2. 软件结构变化
软件从：
传统 UI → Prompt / Agent / Workflow

3. 工具化
LLM 不只是模型，而是：

模型 + 工具 + 数据 + 记忆

---

## 产品方法论

做 AI 产品时关注三个层次：

### 1. 能力层（Model Capability）

核心问题：

模型是否真的能完成任务？

关注：

- reasoning
- tool use
- context length
- reliability

如果能力不稳定，产品体验会非常差。

---

### 2. 系统层（System Design）

AI 产品往往不是单模型，而是系统：

典型结构：

LLM  
+ Tool  
+ Memory  
+ Workflow  
+ Evaluation

系统设计比 prompt 更重要。

---

### 3. 场景层（Use Case）

判断一个 AI 产品是否成立：

核心问题：

**用户是否真的愿意把任务交给 AI？**

常见适合 AI 的场景：

- 信息处理
- 文档生成
- 数据分析
- 自动化操作

---

## 对 Agent 的理解

Agent 本质上是：

**LLM 驱动的自动化系统**

典型结构：

- Planning
- Tool Use
- Memory
- Execution
- Feedback

关键挑战：

1. 稳定性
2. 可控性
3. 成本
4. 评估

---

## 对产品经理角色的理解

AI 产品经理的核心能力：

1. 理解技术边界  
2. 设计系统结构  
3. 找到真实场景  
4. 快速验证  

AI 产品 PM 需要：

- 比传统 PM 更理解技术
- 比工程师更理解用户

---

## 长期关注方向

重点关注：

- AI Agent
- 开发者工具
- 企业效率工具
- AI Native Software

长期目标：

构建真正由 AI 驱动的新型软件系统。
